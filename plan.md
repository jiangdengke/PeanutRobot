# Plan

更新时间：2026-06-01

## 目标

把 PeanutRobot 稳定成可现场交付的机器人控制应用：主流程必须可靠，接口契约必须清晰，现场配置必须可调整，构建发布必须可复现。

## 当前基线

- Android 单模块工程，主入口是 `MainActivity`。
- 已接入擎朗 Peanut SDK，本地 AAR 位于 `app/libs/peanut-sdk-release.aar`。
- 已具备点位读取、点位导航、回充、停止导航、副屏展示、状态上报、屏保锁屏、仓库任务按钮等核心能力。
- 已配置 `v*` tag 触发的 GitHub Actions Debug APK 发布流程。
- 已补齐项目总览文档 `README.md` 和任务跟踪文档 `task.md`。

## 阶段 1：现场稳定性

优先处理会直接影响现场使用的问题。

- 把 WebSocket 地址、HTTP 端口、机器人 ID、任务 ID、SDK 鉴权参数从 `MainActivity` 和 `WebSocketService` 中抽出，改为可配置来源。
- 修复点位选择状态依赖 RecyclerView item view 的问题，改成数据模型级选中状态，避免滚动和复用导致选择丢失。
- 修复未选择点位时“立即出发”按钮被禁用后不恢复的问题。
- 为远程任务回调增加空值保护和状态校验，避免 SDK、充电模块、导航模块未初始化时崩溃。
- 梳理 HTTP 接口响应格式，区分成功、参数错误、执行失败和服务未就绪。
- 明确锁屏解锁交互和默认密码初始化逻辑，避免现场误锁。

## 阶段 2：接口和配置规范化

让外部系统可以稳定对接，减少现场改代码。

- 为 `/robot_task/*` 接口补充固定 JSON 契约和示例。
- 为状态上报 WebSocket 固定字段含义和枚举值。
- 增加运行时配置页或本地配置文件，用于修改服务器地址、任务 ID、速度、锁屏参数。
- 为视频上传和副屏媒体播放增加文件大小、格式、路径校验。
- 统一日志 tag 和关键事件日志，方便现场排查。

## 阶段 3：工程质量

降低后续维护成本。

- 清理 Gradle 依赖重复项，包括 Material、Java-WebSocket、Kotlin Gradle 插件等。
- 评估并逐步移除 `jcenter()` 依赖。
- 把 MainActivity 中的 SDK 初始化、HTTP 路由、导航、锁屏、副屏展示拆分到独立组件。
- 增加基础单元测试和可执行的接口 smoke test。
- 建立 Android 设备或机器人现场验证清单。

## 阶段 4：发布和交付

形成可追踪、可回滚的交付流程。

- 区分 debug、staging、release 构建配置。
- 增加版本号自动校验，避免 tag 与 `versionName` 不一致。
- 在 Release notes 中自动附带接口变更、配置变更和现场验证结果。
- 如需正式交付，补充签名配置和 release APK/AAB 构建流程。

## 风险

- Peanut SDK 授权与运行包名、签名、设备环境强相关，变更前必须确认授权范围。
- 当前多个局域网地址硬编码，换现场网络后会直接影响上报和仓库任务。
- 存储权限和外部存储路径在 Android 版本升级后可能受限制。
- 副屏展示依赖设备存在第二块 Display，普通手机或模拟器无法完整验证。
